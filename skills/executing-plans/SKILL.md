# 执行计划

## 概述

按批次执行实现计划中的任务，每个批次后设置人工检查点以进行审查。

**开始时宣布：** "我正在使用 executing-plans 技能按计划执行实现任务。"

## 执行流程

### 批次执行

将任务分组为逻辑上连贯的批次（通常 3-5 个相关任务），按以下方式执行：

1. 宣布当前批次：展示批次中要完成的任务
2. 执行批次中的每个任务：严格按照计划中的步骤执行
3. 批次完成后：汇总已完成的任务和变更的文件
4. 检查点：询问用户是否准备好继续下一批次

### 每个任务的执行

对于计划中的每个任务：
- 按照任务步骤严格遵循 TDD 流程
- 在每个步骤后验证结果
- 如果遇到意外问题，记录并暂停以重新评估

## 检查点审查

每个批次完成后：
- 列出已完成的任务
- 总结所做的更改
- 展示测试结果
- 询问："这个批次看起来好吗？是否继续下一批次？"

## 错误处理

如果在执行过程中出现问题：
1. 记录实际错误
2. 检查是否与计划有偏差
3. 如果需要调整计划，提出具体修改建议
4. 获得批准后再继续

## 关键规则

- 严格遵循计划 — 不添加计划外的功能
- 每个步骤后验证 — 不要假设成功
- 保持频繁提交 — 每个任务完成后提交
- 使用 conventional commits 格式
- 如果计划有误 — 停下来讨论，不要盲目继续
