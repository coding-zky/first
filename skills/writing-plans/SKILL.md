# 编写实现计划

## 概述

编写全面的实现计划，假设工程师对我们的代码库有零背景且品味堪忧。记录他们需要知道的一切：每个任务要触摸哪些文件、代码、测试、可能需要查阅的文档、如何测试。以小块任务的形式给出整个计划。DRY、YAGNI、TDD、频繁提交。

假设他们是一个熟练的开发者，但对我们的工具集和问题领域几乎一无所知。假设他们不太了解良好的测试设计。

**开始时宣布：**"我正在使用 writing-plans 技能创建实现计划。"

**保存计划到：** `docs/superpowers/plans/YYYY-MM-DD-<功能名>.md`

## 范围检查

如果规范涵盖多个独立子系统，应该在头脑风暴阶段分解为子项目规范。如果没有，建议将其分解为单独的计划 — 每个子系统一个。每个计划应该独立产生可工作、可测试的软件。

## 文件结构

在定义任务之前，规划哪些文件将被创建或修改以及每个文件的职责：

- 设计具有清晰边界和良好定义接口的单元。每个文件应该有一个清晰的职责。
- 更喜欢小而专注的文件，而非做太多事情的大文件。
- 一起变化的文件应该放在一起。按职责而非技术层拆分。
- 在现有代码库中，遵循既定模式。

## 小块任务粒度

**每个步骤一个操作（2-5 分钟）：**
- "编写失败的测试" — 一步
- "运行以确保它失败" — 一步
- "实现最小代码使测试通过" — 一步
- "运行测试并确保通过" — 一步
- "提交" — 一步

## 计划文档头部

**每个计划必须以以下头部开始：**

```markdown
# [功能名称] 实现计划

> **面向代理工作者：** 必需子技能：使用 executing-plans 按任务逐步实现此计划。步骤使用复选框 (`- [ ]`) 语法进行跟踪。

**目标：** [一句话描述构建内容]

**架构：** [2-3 句话描述方案]

**技术栈：** [关键技术/库]

---
```

## 任务结构

```markdown
### 任务 N: [组件名称]

**文件：**
- 创建: `exact/path/to/file.py`
- 修改: `exact/path/to/existing.py:123-145`
- 测试: `tests/exact/path/to/test.py`

- [ ] **步骤 1: 编写失败的测试**

```python
def test_specific_behavior():
    result = function(input)
    assert result == expected
```

- [ ] **步骤 2: 运行测试验证失败**

运行: `pytest tests/path/test.py::test_name -v`
预期: FAIL 带有 "function not defined"

- [ ] **步骤 3: 编写最小实现**

```python
def function(input):
    return expected
```

- [ ] **步骤 4: 运行测试验证通过**

运行: `pytest tests/path/test.py::test_name -v`
预期: PASS

- [ ] **步骤 5: 提交**

```bash
git add tests/path/test.py src/path/file.py
git commit -m "feat: add specific feature"
```
```

## 禁止占位符

每个步骤必须包含工程师需要的实际内容。这些是**计划失败** — 永远不要写：
- "TBD"、"TODO"、"稍后实现"、"填写细节"
- "添加适当的错误处理" / "添加验证" / "处理边界情况"
- "为上述编写测试"（没有实际测试代码）
- "类似于任务 N"（重复代码 — 工程师可能乱序阅读任务）
- 描述做什么但不展示如何做的步骤（代码步骤需要代码块）
- 引用未在任何任务中定义的类型、函数或方法

## 自我审查

编写完完整计划后，以全新视角审视规范并对照检查计划：

1. **规范覆盖：** 浏览规范中的每个部分/需求。能找到实现它的任务吗？列出任何缺口。
2. **占位符扫描：** 搜索计划中的红旗 — 任何来自"禁止占位符"部分的模式。修复它们。
3. **类型一致性：** 后续任务中使用的类型、方法签名和属性名称与之前定义的一致吗？

## 执行移交

保存计划后，提供执行选项：

"计划已完成并保存到 `docs/superpowers/plans/<文件名>.md`。开始按计划逐步执行。

**必需子技能：** 使用 executing-plans 技能按任务逐步实现此计划，每个批次后进行检查点审查。"
